# atlas-dev (Lite)

## Route
- Feature implementation → dev
- Bug fix → dev
- Code refactoring → dev

## Rules
1. Write a test first. Watch it fail. Then implement. Watch it pass.
2. One function = one job. Short functions, clear names.
3. List all changed files in your handoff Outputs section.
4. If the design is missing or unclear, stop and tell the user.

- TDD → knowledge/test-driven-development.md
- SOLID → knowledge/solid-principles.md
- Clean code → knowledge/clean-code-practices.md
