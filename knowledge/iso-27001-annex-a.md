# ISO 27001 Annex A

## Quick Reference
- 93 controls across 4 themes: Organizational, People, Physical, Technological
- Statement of Applicability (SoA): which controls are selected and why
- Risk-based: select controls based on risk assessment
- Certification: external audit confirms ISMS implementation
- Annual surveillance audits, recertification every 3 years

## Deep Dive

### Control Themes
| Theme | Controls | Examples |
|-------|----------|----------|
| Organizational (A.5) | 37 | Policies, roles, asset management, supplier security |
| People (A.6) | 8 | Terms of employment, screening, awareness |
| Physical (A.7) | 14 | Perimeters, equipment security, clean desk |
| Technological (A.8) | 34 | Access rights, cryptography, secure development |

### Key Controls
- **A.5.1**: Policies for information security
- **A.5.7**: Threat intelligence
- **A.5.23**: Information security for cloud services
- **A.6.3**: Information security awareness, education, training
- **A.8.1**: User endpoint devices
- **A.8.5**: Secure authentication
- **A.8.8**: Technical vulnerability management
- **A.8.9**: Configuration management

### Statement of Applicability (SoA)
Document which controls are:
- Implemented (with justification)
- Not implemented (with justification)
- Partially implemented (with plan)

### Implementation Process
1. Define ISMS scope
2. Conduct risk assessment
3. Select controls (Annex A)
4. Implement controls
5. Monitor and measure
6. Internal audit
7. Management review
8. Certification audit
