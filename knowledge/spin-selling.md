# SPIN Selling

## Quick Reference
- **S**ituation: understand the current state
- **P**roblem: identify difficulties and dissatisfactions
- **I**mplication: explore consequences and effects
- **N**eed-payoff: establish the value of a solution
- Use for discovery meetings, requirements elicitation, business case development

## Deep Dive

### SPIN Questions
| Type | Purpose | Example |
|------|---------|---------|
| Situation | Understand current state | "How do you currently handle...?" |
| Problem | Identify pain points | "What difficulties do you have with...?" |
| Implication | Explore consequences | "What happens when...?" / "How does that affect...?" |
| Need-payoff | Establish value | "How would it help if...?" / "What would be the benefit of...?" |

### Why SPIN Works
- Situation questions: necessary but boring (use sparingly)
- Problem questions: reveal pain, create engagement
- Implication questions: amplify pain, create urgency
- Need-payoff questions: let the stakeholder articulate the solution value

### SPIN in Business Analysis
1. **Situation**: understand current process and tools
2. **Problem**: identify what's not working
3. **Implication**: quantify the impact (time lost, errors, cost)
4. **Need-payoff**: let stakeholder describe the ideal solution

### Common Mistakes
- Jumping to solutions before understanding problems
- Asking too many situation questions (boring)
- Not exploring implications (miss the urgency)
- Telling rather than asking (need-payoff)
