# NIST Cybersecurity Framework (CSF)

## Quick Reference
- 6 functions: Identify, Protect, Detect, Respond, Recover, Govern
- Implementation tiers: Partial, Risk-Informed, Repeatable, Adaptive
- Profiles: current state vs target state
- Maps to ISO 27001, NIST 800-53, COBIT
- Risk-based, outcome-focused

## Deep Dive

### Functions
| Function | Purpose | Key Categories |
|----------|---------|----------------|
| Govern | Establish cybersecurity strategy | Governance, risk management strategy |
| Identify | Understand assets and risks | Asset management, risk assessment |
| Protect | Implement safeguards | Access control, awareness, data security |
| Detect | Find cybersecurity events | Anomalies, events, continuous monitoring |
| Respond | Take action on events | Response planning, communications, mitigation |
| Recover | Restore capabilities | Recovery planning, improvements |

### Implementation Tiers
| Tier | Description |
|------|-------------|
| 1 - Partial | Ad hoc, reactive |
| 2 - Risk-Informed | Risk-aware but not organization-wide |
| 3 - Repeatable | Formal policies, regularly updated |
| 4 - Adaptive | Continuous improvement, threat-adaptive |

### CSF Profile
- **Current Profile**: current cybersecurity outcomes
- **Target Profile**: desired cybersecurity outcomes
- **Gap analysis**: difference between current and target
- **Roadmap**: plan to close gaps

### Using CSF
1. Identify critical assets and services
2. Assess current profile
3. Define target profile
4. Conduct gap analysis
5. Prioritize improvements
6. Implement changes
7. Monitor and review
